Modèle Conceptuel de Données

Explication des éléments

Jeux-vidéos

  1. ID : Clé primaire de l'élément, permet d'identifier les différents jeux même si les noms sont les mêmes.
  2. Nom : Nom attribué du jeu.
  3. Date de sortie : Date de la sortie du jeu, la première version disponible au public si il en existe plusieurs.
  4. Note JVC : Note attribuée au jeu par le site référence jeuxvideos.com.
  5. PEGI : Le système d'évaluation européen des jeux vidéo qui permet de les classer par âges pour informer les consommateurs.
  6. RECOMPENSE : L'identifiant vers la récompense attribuée au jeu aux Game Awards, si il en a reçu.

Editeur

  1. Nom : Clé primaire de l'élément, correspond aun nom de l'entreprise, il est forcément unique car déposé.
  2. Date de création : Correspond à la date de création référencée de l'entreprise, le mois et l'année uniquement si le jour n'est pas indiqué
  3. Siège social : Ville ou est situé l'entreprise ou son domicile juridique.
  4. Jeu principal : Le plus gros jeu de l'entreprise, défini par le nombre de ventes.
  5. Site : Le site web de l'entreprise si il existe.

Studio de développement

  1. Nom : Clé primaire de l'élément, correspond aun nom de l'entreprise, il est forcément unique car déposé.
  2. Date de création : Correspond à la date de création référencée de l'entreprise, le mois et l'année uniquement si le jour n'est pas indiqué
  3. Siège social : Ville ou est situé l'entreprise ou son domicile juridique.
  4. Jeu principal : Le plus gros jeu de l'entreprise, défini par le nombre de ventes.

Game Awards

  1. ID : Clé primaire de l'élément, permet d'identifier les différents prix même si les libellés sont les mêmes.
  2. Année : Année à laquelle la récompense a été décernée.
  3. Libellé : Le nom de la récompense obtenue aux Game Awards.

Thème

  1. IDJeu : Clé primaire de l'élément (étrangère), l'identifiant ID d'un jeu vidéo.
  2. Thème : Correspond aux thèmes de l'oeuvre, à ne pas confondre avec le genre.
  3. Dvlpmt Durable : Explicite l'abord du développement durable dans l'oeuvre.

Genre

  1. IDGenre : Clé primaire de l'élément, permet d'identifier les différents genre de jeux.
  2. Type : Un genre en particulier (exemple disponible dans BDD).

Relations

  1. développé par :
    "JEU VIDEO (1,1) -> STUDIO DEV (0,N)"
    relie un studio avec son ou ses jeux.
  2. édité par :
    "JEU VIDEO (1,1) -> EDITEUR (0,N)"
    relie un éditeur avec son jeu. La relation existe directement dans les éléments grace à l'attribut "Jeu principal"
  3. possède :
    "EDITEUR (1,N) -> STUDIO DEVELOPPEMENT"
    Un éditeur relié a son ou ses studios.
  4. a obtenu :
    "JEU VIDEO (0,N) -> GAME AWARDS (1,1)"
    Relie un jeu-vidéo à sa récompense si il en a obtenue une, relation intégrée au premier élément.
  5. a pour thème :
    "JEU VIDEO (1,1) -> THEME (1,N)"
    Relie un jeu-vidéo à son thème, relation intégrée au premier élément.
  6. a pour genre :
    "THEME (1,N) -> GENRE (0,N)"
    Relie un thème (et donc un jeu vidéo) avec un ou plusieurs genres, mais il se peut qu'un genre ne soit pas représenté dans la base.
